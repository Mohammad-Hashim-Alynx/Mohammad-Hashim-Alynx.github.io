IntelliFlow™

Municipal Water Pump Automation & Protection System (ESP32-Based)

A failsafe, offline-first pump control system engineered for municipal, agricultural, and industrial water infrastructure.

1. Project Overview

IntelliFlow is a production-grade water pump automation system designed for critical water infrastructure where reliability is non-negotiable. It eliminates the failure modes of float switches and timer-based controllers by enforcing deterministic, sensor-verified operation.

The system protects pumps from dry-running, prevents tank overflow, handles intermittent water sources, and rejects false triggers caused by turbulence or debris. It operates fully autonomously without cloud dependency and is suitable for 24/7 deployment.

2. System Architecture

The architecture is centered on an ESP32 microcontroller and follows a strict separation of concerns. Conductivity-based sensing feeds a deterministic finite state machine, which drives an opto-isolated pump relay stage. 

Sensing Layer → FSM (ESP32 Core 0) → Isolated Relay → Pump
Diagnostics & Logging → ESP32 Core 1

3. Water Detection & Sensor Subsystems

  • Electrode 1 (Source/Inlet): Prevents pump start without water.
  • Electrode 2 (Flow Verification): Confirms real flow after pump start.
  • Electrode 3 (Tank Full): Prevents overflow.

Each electrode uses an excitation-and-sense technique with ADC-based threshold detection. Software debouncing and moving averages eliminate noise from turbulence and EMI.

4. Control Logic & Finite State Machine

  1. STATE_WAITING_FOR_WATER: Requires stable source water.
  2. STATE_STARTUP_STABILIZATION: 3-minute source stabilization delay.
  3. STATE_PUMP_RUNNING: Active pumping with continuous validation.
  4. STATE_FAILURE_COOLDOWN: 20-minute enforced cooldown after faults.

The FSM is fully non-blocking and driven by millis(), ensuring continuous responsiveness and fault detection.

5. Safety & Protection Mechanisms

  • Two-stage dry-run protection with source + flow verification
  • Tank overflow prevention with confirmation delay
  • Mandatory cooldown to prevent rapid cycling
  • Reset-cause diagnostics for field troubleshooting

6. Firmware Architecture

Firmware is modular and industrial in structure, separating sensing, logic, safety, I/O control, and diagnostics. The system is fully offline-first and cloud-independent.

7. Installation & Field Deployment

The controller is housed in an IP66 enclosure with optically isolated pump control. Electrode placement follows strict hydraulic and safety guidelines for reliable detection.
