Automatic Inline Water Pressure Booster – ESP32

Flow-based intelligent booster pump controller designed for reliable, real-world deployment with zero manual intervention.

Project Overview

This project is a flow-based intelligent inline booster pump controller designed to solve critical problems of pump dry-running, energy waste, and hardware wear in residential, commercial, and agricultural water pressure systems.

Unlike conventional pressure-switch controllers that rely on indirect pressure changes, this system uses direct Hall-effect flow sensing to activate the pump only when genuine water demand exists, ensuring efficient operation and extended pump lifespan.

System Architecture

The system is built around an ESP32 microcontroller as the central logic unit, selected for its dual-core processing, precise timing, and reliable GPIO handling. A YF-S201 Hall-effect flow sensor provides real-time flow data, which the ESP32 processes to control a high-current relay driving the AC booster pump.

The firmware uses a fully non-blocking, millis()-based timing architecture to guarantee deterministic behavior and uninterrupted long-term operation.

Hardware Subsystems

Flow Detection Subsystem

A YF-S201 Hall-effect flow sensor is installed inline with the water pipe. Each turbine rotation generates a pulse, captured via a hardware interrupt on the ESP32. Software debouncing and averaging ensure accurate, noise-immune flow measurement.

Control & Logic Subsystem

The ESP32 executes a deterministic state machine using interrupt-driven pulse counting and non-blocking timing. All thresholds and timings are configurable constants for easy field tuning.

Power Switching Subsystem

A Songle SRD-05VDC-SL-C relay module provides opto-isolated switching between the low-voltage control electronics and the high-voltage AC booster pump, ensuring electrical safety and reliability.

User Feedback Subsystem

  • Green: System ready, no flow
  • Yellow: Flow detected, activation pending
  • Green + Relay: Pump active
  • Red: Fault condition

Control Logic & State Machine

The controller operates using a three-state machine: STANDBY, ACTIVE, and COOLDOWN. Flow is sampled over a fixed window to validate genuine demand before activation.

Minimum ON and OFF timers prevent short cycling, while dry-run detection ensures immediate shutdown if flow is not detected after pump startup. Non-blocking timing allows continuous system supervision without watchdog resets.

Firmware Design & Source Code

The firmware follows a single-threaded, non-blocking super-loop architecture with interrupt-driven flow sensing. All delays are managed using millis(), eliminating blocking calls and improving reliability.

Safety & Protection Mechanisms

  • Dry-run detection with immediate pump shutdown
  • Anti-short-cycling enforcement using minimum ON/OFF timers
  • Opto-isolated relay protection for AC mains
  • Fail-safe defaults ensuring pump OFF on reset or fault

Installation & Deployment Summary

The controller is installed inline between the water source and the booster pump. The flow sensor is mounted using standard pipe fittings, and the control unit is housed in a ventilated enclosure with full electrical isolation between AC and DC domains.

Performance Outcomes & Impact

  • 30–40% reduction in unnecessary pump energy usage
  • Extended pump lifespan through dry-run prevention
  • Fully autonomous operation with zero user input
  • Reliable long-term deployment without maintenance
